# Epistory 打字自动化工具
《Epistory - Typing Chronicles（纸境奇缘）》战斗半自动辅助工具，纯外部视觉方案，无注入、无修改游戏内存，安全稳定。

## ✨ 功能特点
- 🚀 **零侵入**：纯截图+键盘模拟，不读取/修改游戏内存，无封号风险
- ⚡ **高速识别**：基于 RapidOCR ONNX 推理引擎，CPU 即可实时运行
- 🎯 **精准识别**：针对游戏场景优化预处理，单词识别准确率98%+
- 🤖 **拟人化打字**：随机间隔模拟人手速，无机械行为特征
- 🛡️ **安全防护**：仅游戏前台时生效，切出窗口自动暂停，避免误打字
- 🔧 **高度可配置**：识别区域、打字节奏、置信度均可自由调整

## 🛠️ 安装步骤
### 1. 环境要求
- Windows 10/11（游戏运行平台）
- Python 3.8 ~ 3.12
- 游戏设置为**窗口化模式**（推荐分辨率 1280×720 / 1920×1080）

### 2. 安装依赖
进入项目目录，执行：
```bash
pip install -r requirements.txt
```

> 首次运行会自动下载 RapidOCR 英文轻量模型（约10MB），保持联网即可。

## 🚀 使用方法
1. 启动《Epistory》游戏，保持窗口化、可见（不要最小化）
2. 运行主程序：
   ```bash
   python main.py
   ```
3. 控制台提示「已定位游戏窗口」即为启动成功
4. 手动控制角色移动探索，遇敌按空格进入打字模式，工具会自动识别单词并输入
5. 切出游戏窗口自动暂停打字；按 `Ctrl+C` 可完全终止脚本

## ⚙️ 参数调整说明
所有参数均可在 `main.py` 顶部的「参数配置区」修改：

### 识别精度调整
| 参数 | 作用 | 调整建议 |
|------|------|----------|
| `OCR_CONF_THRESHOLD` | OCR置信度阈值 | 误识别多就调高到0.85~0.9；漏识别就调低到0.7~0.75 |
| `MIN_WORD_LEN` | 最短有效单词长度 | 单字母误识别多就调到3 |

### 识别区域调整（最常用）
| 参数 | 作用 | 调整方法 |
|------|------|----------|
| `ROI_TOP_RATIO` | 顶部裁剪比例 | 顶部UI干扰就增大；高处单词被裁就减小 |
| `ROI_BOTTOM_RATIO` | 底部截止比例 | 底部UI干扰就减小；近处单词被裁就增大 |
| `ROI_LEFT_RATIO` / `ROI_RIGHT_RATIO` | 左右裁剪比例 | 两侧有干扰就对应调整 |

> 💡 **调试技巧**：将 `DEBUG_SHOW_ROI` 设为 `True`，运行后会弹出「OCR识别区域预览」窗口，边看边调参数，调到「所有单词完整在区域内、无多余UI」即可。

### 打字节奏调整
| 参数 | 作用 | 调整建议 |
|------|------|----------|
| `CHAR_MIN_DELAY` / `CHAR_MAX_DELAY` | 单字符输入间隔 | 想更拟人就调大到0.1~0.2；想更快就调小 |
| `WORD_MIN_DELAY` / `WORD_MAX_DELAY` | 单词间停顿 | 同上 |
| `SCAN_INTERVAL` | 画面扫描间隔 | 响应慢就调小到0.08；CPU占用高就调大到0.15 |

## ❓ 常见问题
### Q: 识别不准/漏识别怎么办？
1. 开启 `DEBUG_SHOW_ROI` 检查识别区域是否完整包含单词
2. 适当降低 `OCR_CONF_THRESHOLD`
3. 提高游戏内亮度和对比度，让文字更清晰
4. 检查游戏是否为窗口化，无边框窗口可能定位不准

### Q: 误识别太多/乱打字怎么办？
1. 提高 `OCR_CONF_THRESHOLD` 到0.85以上
2. 增大裁剪比例，裁掉更多UI干扰区域
3. 调大 `MIN_WORD_LEN` 到3，过滤单字母误识别

### Q: 打字到其他窗口了怎么办？
工具自带前台检测，只有游戏窗口处于最前端时才会打字，切出自动暂停。如果还是出现问题，检查游戏窗口标题是否包含 `Epistory` 关键词。
